I modified the actual calls for obvious reasons- can't have people spamming The Met's ticket printers through my site.

The original the printer status table was populated externally by first calling a 'get' to the IT Department's printer network for the printer list as raw data. That was then parsed into JSON, which was then used to populate the table. Clicking any of the printers from the status table would fire off a 'post' asking the test printer to print a test ticket. For this preview I've collapsed that down so the table populates from another html page without calling the server.
